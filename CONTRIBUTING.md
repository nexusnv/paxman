# Contributing to Paxman

Thanks for your interest in contributing! The full contributing guide is
published on the Paxman documentation site:

**[Contributing guide →](https://paxman.readthedocs.io/en/latest/contributing/)**

That page covers the contribution workflow, the ADR-driven process for
significant changes, the local dev setup, the test strategy, and the
code of conduct.

## Quick links

- **Bug reports & feature requests:** [github.com/nexusnv/paxman/issues](https://github.com/nexusnv/paxman/issues)
- **Discussions & design questions:** [github.com/nexusnv/paxman/discussions](https://github.com/nexusnv/paxman/discussions)
- **Pull request template:** [.github/PULL_REQUEST_TEMPLATE.md](.github/PULL_REQUEST_TEMPLATE.md)

## Code of conduct

Paxman follows the [Contributor Covenant v2.1](CODE_OF_CONDUCT.md). By
participating, you are expected to uphold this code.
