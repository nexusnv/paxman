# Security policy

> **TL;DR:** Report security vulnerabilities to `security@nexusnv.net`
> (encrypted, GPG key on request). Do **not** open a public GitHub issue
> for security reports.

The full security policy — threat model, PII handling rules, secrets-by-reference
guidance, and the disclosure timeline — is published on the Paxman
documentation site:

**[Security policy →](https://paxman.readthedocs.io/en/latest/security/)**

## Reporting a vulnerability

Email: **`security@nexusnv.net`**

Please include:

1. A description of the vulnerability and its impact.
2. Steps to reproduce or a proof-of-concept.
3. The version of Paxman affected.
4. Whether you intend to disclose publicly (and on what timeline).

We aim to acknowledge security reports within 2 business days and to
issue a fix or a mitigation within 30 days, depending on severity.

## Supported versions

| Version | Supported |
|---|---|
| 1.0.x (latest) | ✅ Active |
| < 1.0.0 | ❌ No longer supported |
