Why Paxman

Normalization without silent failure

Existing approaches force you to choose between brittle hand-rolled pipelines, unverifiable LLM calls, and schema-only validators that can't extract a thing.

What goes wrong today

Failure modeCost
Hand-rolled per-source pipelinesHigh maintenance
"Just call an LLM" — no evidence, no replayUntraceable
Schema-validators without extractionWrite extractors per source
Auto-schema that hallucinates fieldsData corruption
Monolithic document servicesVendor lock-in

The Paxman philosophy

No contract, no normalization. Paxman cannot normalize without a target contract. The contract describes what, not how — planning and execution decide how to satisfy requirements.

Paxman does not guess silently. Every inferred value carries evidence, confidence, and resolution metadata. Unresolved fields are always explicit with a reason.

Determinism is required. Same inputs + contract + configuration must yield the same plan and replayable result. Every artifact carries a replay hash (SHA-256) for tamper detection.

Cost is a first-class concern. Paxman prefers the cheapest sufficient path that still satisfies accuracy policy.

Capabilities

Everything you need, nothing you don't

A tiny public API backed by a principled seven-subsystem architecture.

Contract-Driven

You bring the contract — Pydantic, JSON Schema, OpenAPI, or Dict DSL. Paxman canonicalizes it and plans per-field extraction. It never owns your schema.

Deterministic Planning

Each required field gets its own plan through a 7-step heuristic chain: explicit evidence, regex, structured lookup, derived computation, local inference, remote inference, or UNRESOLVED.

Evidence-Backed Output

Every resolved fact carries provenance — capability ID, source span, model ID. Inferred values are distinguishable from directly observed ones.

Deterministic Replay

Replay from the artifact alone — no recomputation. SHA-256 replay hash guards against tampering. Version checks prevent cross-version replay mismatches.

Budget Awareness

Hard caps on cost (USD), latency (ms), remote inference calls, and total capability invocations. Soft policy on confidence floors, currency rules, and remote inference.

Extensible by Design

Public SPIs for ContractAdapter, Capability, Heuristic, and InferenceProvider. Register your own adapters and capabilities — Paxman does the orchestration.

Code

See it in action

Three examples that cover the full workflow — normalize, replay, and extend.

# === Basic normalization === import paxman from decimal import Decimal from pydantic import BaseModel class Invoice(BaseModel): supplier_name: str total_amount: float currency_code: str line_items: list[LineItem] result = paxman.normalize( input_data=raw_invoice_bytes, contract=Invoice, budget=paxman.Budget(max_total_cost_usd=Decimal("0.10")), policy=paxman.Policy(allow_remote_inference=True), ) print(result.normalized_data) # matches the Invoice shape print(result.unresolved_fields) # fields Paxman could not resolve print(result.replay_hash) # deterministic signature # Replay later from the artifact alone rehydrated = paxman.replay(result, contract=Invoice) assert rehydrated == result # byte-equal
# === Budget & Policy controls === import paxman from decimal import Decimal from pydantic import BaseModel, Field class Invoice(BaseModel): supplier_name: str = Field(..., description="Supplier name") total_amount: float = Field(..., description="Total amount") currency_code: str = Field(..., description="ISO-4217 code") artifact = paxman.normalize( input_data=raw_invoice, contract=Invoice, budget=paxman.Budget( max_total_cost_usd=Decimal("0.10"), max_total_latency_ms=5_000, max_remote_inference_calls=2, ), policy=paxman.Policy( allow_remote_inference=True, confidence_floor=0.80, unresolved_acceptable=False, ), ) print(artifact.status) # Status.SUCCESS or PARTIAL_SUCCESS print(artifact.normalized_data) assert artifact.unresolved_fields == []

Applications

Built for real-world pipelines

Paxman fits where auditable, evidence-backed normalization meets production requirements.

Invoice & Procurement

Normalize invoices, quotations, and purchase orders across suppliers and currencies. Compare offers side-by-side with traceable field-level evidence.

Agentic Ingestion & Discovery

Auditable, evidence-backed extraction for RAG pipelines and agent workflows. Every value carries provenance — no silent hallucinations in downstream systems.

Document Understanding SaaS

Wrap Paxman inside your service without giving up replay or evidence. Your customers get traceable results; you get a predictable, extensible normalization core.

Multi-Source Data Pipelines

Normalize email, OCR, CSV, and API payloads into one canonical schema. Paxman's field-centric planning adapts to each source automatically.
